## 启动命令清单（无 Docker + uv）
1. 后端环境初始化
- `cd backend`
- `uv venv`
- `uv pip install -r requirements.txt`

2. 启动 API
- `cd backend`
- `uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000`

3. 启动 Worker
- `cd backend`
- `uv run celery -A app.worker.celery_app:celery_app worker --loglevel=info --pool=solo`

4. 启动前端
- `cd frontend`
- `npm install`
- `npm run dev`

5. Redis 前置检查（本机服务）
- 确保 `REDIS_URL=redis://127.0.0.1:6379/0` 可连接
- 启动脚本在连接失败时直接退出并提示

## 最小接口契约（MVP）
1. `POST /api/auth/register`
- 请求：`{ username, password }`
- 响应：`{ user_id, username }`

2. `POST /api/auth/login`
- 请求：`{ username, password }`
- 响应：`{ access_token, token_type }`

3. `POST /api/tasks/submit`
- Header：`Authorization: Bearer <token>`
- 请求：`{ code: string }`
- 响应：`{ task_id, status: "PENDING" }`

4. `GET /api/tasks/{task_id}`
- Header：`Authorization: Bearer <token>`
- 响应：`{ task_id, status, error_message? }`

5. `GET /api/tasks/{task_id}/result`
- Header：`Authorization: Bearer <token>`
- 响应：`{ task_id, status, result: { counts, probabilities } }`

## 风险-应对矩阵
1. 风险：本机 Redis 未启动或端口冲突
- 影响：任务无法入队，链路中断
- 应对：启动前健康检查 + 明确错误提示 + README 故障排查

2. 风险：Qibo 脚本执行超时或异常
- 影响：Worker 堵塞或任务失败率高
- 应对：执行超时、异常捕获、任务状态落库、错误信息回传

3. 风险：脚本执行安全边界不足
- 影响：存在执行逃逸风险
- 应对：导入白名单、禁用系统调用、限制执行上下文

4. 风险：单人开发阶段联调成本高
- 影响：前后端接口反复改动
- 应对：冻结最小接口契约，先 mock 后联调

## 质量门禁
- 严格 YAGNI：不引入 RBAC/计费/拖拽电路
- 严格 KISS：单 Redis、单 Worker、单 SQLite
- 可观测最小集：任务状态、执行时长、错误原因
