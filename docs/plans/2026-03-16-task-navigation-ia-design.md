# Task Navigation IA Design

## 目标

为任务域（`/tasks/*`）提供统一的导航与路径感知，解决“用户不清楚当前所在位置”与“默认跳转语义不清”问题。

## 范围（P0）

- 新增任务域全局顶部导航（模块切换 + 当前模块高亮）。
- 新增全局面包屑（`任务 / 当前模块`）。
- 将 `/tasks` 默认入口由 `/tasks/circuit` 调整为 `/tasks/center`。
- 任务中心新增“帮助文档”入口。
- 标题文案增强，让页面目标可读。

## 信息架构决策

- 一级：`任务`
- 二级模块：
  - `任务中心`（`/tasks/center`）
  - `图形化编程`（`/tasks/circuit`）
  - `代码提交`（`/tasks/code`）
  - `帮助文档`（`/tasks/help`）

理由：`任务中心`承担任务查询与结果诊断，最适合作为任务域默认入口。

## 技术方案

1. 引入 `TasksWorkspaceLayout` 作为 `/tasks` 父路由壳层，负责：
   - 顶部导航
   - 当前模块解析
   - 面包屑渲染
   - `Outlet` 内容承载
2. 路由从平铺改为嵌套：
   - `/tasks` → index redirect to `center`
   - `/tasks/center|circuit|code|help`
3. 任务中心页头增加帮助入口；三类任务页标题补充“模块目的”。

## 风险与回滚

- 风险：路由结构变更导致现有测试断言失效。
- 控制：同步更新 `workbench-route` 与新增布局测试。
- 回滚：仅涉及前端路由与组件，单次回滚到前一提交即可恢复。

## 验收标准

- 访问任一 `/tasks/*` 页面都能看到统一顶部导航与面包屑。
- `/tasks` 进入后默认显示任务中心。
- 任务中心可一键进入帮助文档页。
- 现有受保护路由行为不回归（未登录仍跳转登录）。
